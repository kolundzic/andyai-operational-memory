# Release Announcement Copy

## X / Threads / LinkedIn short version

We’re releasing **AndyAI Operational Memory v1.0**.

This project explores a simple idea:

**semantic memory is not enough.**

AI systems need a memory layer that can do more than retrieve similar text.
They need structure, lifecycle, trust, promotion, signed exports, and replayable artifacts.

AndyAI Operational Memory is a trust-aware memory foundation for agentic workflows:
- semantic retrieval
- promotion gates
- evidence/trust metadata
- signed trust bundles
- replay manifests
- CI release discipline

Core formula:

**Memory + State + Direction + Trust + Action**

## Slightly longer version

Today we’re publishing **AndyAI Operational Memory v1.0** — a public foundation for thinking about memory not as a convenience feature, but as operational infrastructure.

Most AI memory discussions stop at embeddings and retrieval.
That’s useful, but incomplete.

This repo pushes further:
- memory records with lifecycle
- dedup and promotion gates
- trust/evidence metadata
- signed export bundles
- replay-ready manifests
- release discipline through CI

The core thesis is simple:

**Semantic memory helps AI find things.  
Operational memory helps AI build with continuity.**
